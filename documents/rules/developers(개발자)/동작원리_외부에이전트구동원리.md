# 외부 에이전트 소스 통합 (External Agent Sources) 구현 계획

> 작성일: 2026-01-19
> 상태: ✅ **구현 완료** (2026-01-19 02:27)

## 목표

Agent Protocol 표준을 따르는 외부 에이전트 서버들을 `agent-middleware`에 연결하여, 
내부 에이전트와 외부 에이전트를 **통합 관리**하는 기능을 구현합니다.

## 배경

- 사용자가 다양한 외부 에이전트 서버(Agent Protocol 표준)를 쉽게 연결
- 내부/외부 에이전트를 하나의 UI에서 통합 관리
- 에이전트 생태계 확장 지원 (오픈소스 배포 목적)

---

## 실행 시나리오: 계산기 에이전트 예시

사용자가 "333*3를 계산해줘"라고 질문했을 때의 전체 실행 흐름:

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 1. 웹 UI (localhost:3000)                                                       │
│    사용자가 "계산기 에이전트" 선택 → "333*3를 계산해줘" 입력                       │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │ POST /runs/stream
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 2. agent-middleware (localhost:8002)                                            │
│    ┌─────────────────────────────────────────────────────────────────────────┐  │
│    │ 동적 생성된 ReAct 그래프 실행                                            │  │
│    │ - graph_id: external:external_calculator_agent                          │  │
│    │ - system_prompt: "당신은 수학 계산 전문 AI..."                            │  │
│    │ - model: .env의 MODEL (예: lmstudio/openai/gpt-oss-20b)                  │  │
│    │ - tools: [calculator]                                                   │  │
│    └─────────────────────────────────────────────────────────────────────────┘  │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 3. LLM 호출 (call_model 노드)                                                   │
│    - runtime.context.model 사용 (.env MODEL 환경변수 자동 로드)                  │
│    - LLM이 질문 분석 → "계산이 필요하다" → calculator 도구 호출 결정              │
│    - response.tool_calls = [{"name": "calculator", "args": {"expression": "333*3"}}] │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 4. 도구 실행 (tools 노드 - DynamicGraphFactory가 생성한 동적 도구)               │
│    ┌─────────────────────────────────────────────────────────────────────────┐  │
│    │ create_dynamic_tool()로 생성된 도구가 HTTP 요청 전송:                     │  │
│    │                                                                          │  │
│    │   POST http://host.docker.internal:8003/tools/calculator/execute        │  │
│    │   Body: {"expression": "333*3"}                                         │  │
│    │                                                                          │  │
│    │   ※ Pydantic args_schema를 동적 생성하여 파라미터 형식 보장               │  │
│    └─────────────────────────────────────────────────────────────────────────┘  │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 5. agent-external (localhost:8003)                                              │
│    /tools/calculator/execute 엔드포인트 실행                                     │
│    - eval("333*3") → 999                                                        │
│    - 응답: {"success": true, "expression": "333*3", "result": 999}              │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 6. 다시 LLM 호출 (call_model 노드)                                              │
│    - 도구 결과를 받아서 최종 답변 생성                                            │
│    - "333 × 3 = 999입니다"                                                      │
└───────────────────────────────────┬─────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│ 7. 웹 UI에 응답 스트리밍                                                        │
│    "333 × 3 = 999입니다"                                                        │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 핵심 포인트

| 단계 | 위치 | 역할 |
|------|------|------|
| 1-2 | agent-middleware | 동적 그래프 실행, LLM 호출 |
| 3-4 | agent-middleware | 도구 호출 결정 & HTTP 요청 |
| 5 | **agent-external** | 실제 계산 수행 (eval) |
| 6-7 | agent-middleware | 결과 정리 & 응답 스트리밍 |

**중요**: 
- 계산 자체는 `agent-external (8003)`에서 수행됨
- LLM 모델은 `agent-middleware/.env`의 `MODEL` 환경변수 사용
- Docker 환경에서는 `host.docker.internal`로 호스트 접근

## 구현 상세

### 1. 설정 파일 확장

**파일**: `agents.json`

```json
{
  "graphs": {
    "resoning": "./agents/agent_reason/graph.py:graph",
    "resoning_hitl": "./agents/agent_reason_hitl/graph.py:graph",
    "sub_agent": "./agents/agent_subagent/graph.py:graph",
    "research_summary": "./agents/agent_research_summary/graph.py:graph"
  },
  "external_sources": [
    {
      "url": "http://host.docker.internal:8003",
      "name": "Local Test Agents",
      "enabled": true
    }
  ]
}
```

> ⚠️ **Docker 환경 주의**: `localhost` 대신 `host.docker.internal` 사용

---

### 2. 신규 서비스: ExternalSourceService

**파일**: `src/agent_server/services/external_source_service.py`

외부 Agent Protocol 서버와 통신하는 서비스:
- `fetch_agents(url)`: 에이전트 목록 조회
- `fetch_agent_config(url, agent_id)`: 에이전트 설정 조회
- `execute_tool(endpoint, params)`: 외부 도구 실행

---

### 3. 신규 서비스: DynamicGraphFactory  

**파일**: `src/agent_server/services/dynamic_graph_factory.py`

외부 에이전트 설정에서 LangGraph 동적 생성:
- `create_dynamic_tool(tool_def)`: REST API 도구를 StructuredTool로 변환
- `create_graph(config)`: ReAct 스타일 그래프 동적 생성

**핵심 구현**:
- Pydantic `create_model()`로 동적 args_schema 생성
- `runtime.context.model` 사용으로 내부 에이전트와 동일한 모델 패턴

---

### 4. LangGraphService 수정

**파일**: `src/agent_server/services/langgraph_service.py`

`_load_external_sources()` 메서드 추가:
1. `agents.json`의 `external_sources` 읽기
2. 각 소스에서 에이전트 목록 가져오기
3. DynamicGraphFactory로 그래프 동적 생성
4. 레지스트리 및 캐시에 등록

---

### 5. AgentRegistryService 수정

**파일**: `src/agent_server/services/agent_registry_service.py`

`RegisteredAgent`에 필드 추가:
- `source_type: Literal["internal", "external"]`
- `source_url: str | None`

## 런타임 리로드 (2026-01-20 추가)

서버 재시작 없이 외부 에이전트를 리로드할 수 있습니다.

### 방법 1: 웹 UI 버튼

에이전트 선택 화면(`http://localhost:3000/?apiUrl=http://localhost:8002`)에서 **"🔄 외부 에이전트 새로고침"** 버튼 클릭

### 방법 2: API 직접 호출

```bash
curl -X POST http://localhost:8002/external-sources/reload
```

응답 예시:
```json
{
  "success": true,
  "loaded_count": 3,
  "message": "Successfully reloaded 3 external agents"
}
```

### 사용 시나리오

1. `agent-middleware`를 먼저 시작하고, `agent-external`를 나중에 시작한 경우
2. `agent-external`에 새 에이전트를 추가한 후 반영이 필요한 경우
3. `agents.json`의 `external_sources` 설정을 변경한 경우

## 주의사항 (트러블슈팅)

### Docker 환경에서 localhost 접근 불가

**문제**: Docker 컨테이너 내에서 `localhost`는 컨테이너 자체를 가리킴

**해결**: `host.docker.internal` 사용
- `agents.json`: `"url": "http://host.docker.internal:8003"`
- `agent-external/main.py`: 도구 엔드포인트도 `host.docker.internal` 사용

### 도구 파라미터 전달 오류

**문제**: `**kwargs` 사용 시 LLM이 파라미터 형식을 잘못 인식

**해결**: Pydantic `create_model()`로 동적 `args_schema` 생성
```python
DynamicArgsSchema = create_model(f"{tool_def.name}_args", **field_definitions)
return StructuredTool.from_function(..., args_schema=DynamicArgsSchema)
```
